# Phase 3 Plan 1: Investigate All ANE Op Alternatives

## Objective

Systematically test every MIL op needed for fused training programs, including `mb.*` prefixed variants, basic untested ops, conv-based decompositions, and sequence length boundaries. Determine the maximum set of ANE-compatible operations.

## Context

@docs/ane_constraints.md — Empirical results showing 8 ops rejected
@/Users/nat/dev/ANE/training/stories_mil.h — Reference code using 14 distinct MIL ops
@examples/test_ane_constraint.rs — Existing test infrastructure with subprocess isolation

## Tasks

### Task 1: Test `mb.*` prefixed op variants
- **Type**: auto
- **Description**: The ANE may accept `mb.matmul`, `mb.softmax`, `mb.layer_norm`, `mb.concat`, `mb.reduce_sum` as namespace-qualified ops that the standard names don't support. Also test `mb.transpose`, `mb.reshape`, `mb.slice_by_size`. Write new test functions in `test_ane_constraint.rs` for each.
- **Verification**: Each test produces clear PASS (compiles+evals) or FAIL (error message)
- **Done**: 10+ mb.* variant tests with results

### Task 2: Test remaining basic ops (sub, pow, transpose, reshape, slice_by_size)
- **Type**: auto
- **Description**: Test the basic ops that stories_mil.h uses but we haven't tested yet: `sub`, `pow`, `transpose`, `reshape`, `slice_by_size`. These are needed for backward pass (sub), RMSNorm (pow), attention (transpose, reshape), and backward taps (slice_by_size).
- **Verification**: Each test produces clear PASS or FAIL
- **Done**: 5 basic op tests with results

### Task 3: Test conv-based decompositions for rejected ops
- **Type**: auto
- **Description**: If mb.* variants don't work, test whether rejected ops can be decomposed into conv-based alternatives:
  - `reduce_sum` → conv with all-ones kernel (sum reduction via 1x1 conv)
  - `softmax` → exp/sum decomposition (if we have exp)
  - `matmul` → reshape + conv (linear projection via conv1x1 already works)
  - `concat` → multi-output programs (we know this works)
  - `layer_norm` → skip, use manual RMSNorm or ANEMLL trick (if concat works)
- **Verification**: Each decomposition compiles and produces correct-ish output
- **Done**: Decomposition test results for each rejected op

### Task 4: Test sequence length boundary
- **Type**: auto
- **Description**: We know seq=16 works and seq=32 produces nan/inf. Test seq=20, 24, 28, 32, 48, 64 with a simple sigmoid program (no weights) to find the exact boundary. Also test seq=16 with conv+weights to confirm it's seq-dependent not op-dependent.
- **Verification**: Clear pass/fail for each seq length
- **Done**: Sequence length boundary documented

### Task 5: Document findings and commit
- **Type**: auto
- **Description**: Update `docs/ane_constraints.md` with all new findings. Create Phase 3 summary. Update STATE.md with blockers and next steps.
- **Verification**: Documentation committed
- **Done**: Complete ANE op compatibility table with all tested variants

## Success Criteria

1. Every op needed for fused training is either confirmed working or has a documented decomposition
2. Sequence length boundary precisely identified
3. Clear path forward for Phase 3 fused MIL generators (what ops can we use?)
4. All findings documented in `docs/ane_constraints.md`
