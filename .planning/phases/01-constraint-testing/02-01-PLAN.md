# Phase 2 Plan 1: Run Full Constraint Suite, Analyze Results, Update Docs

## Objective

Execute all 30+ ANE constraint tests, capture results, compare against Orion paper predictions, and update constraint documentation with empirical findings.

## Context

@docs/SESSION_ANE_CONSTRAINT_TESTING.md — Constraint coverage map and test names
@docs/ane_constraints.md — Existing constraint documentation (needs empirical updates)
@examples/test_ane_constraints.rs — Orchestrator binary
@examples/test_ane_constraint.rs — Worker binary with all test functions

## Tasks

### Task 1: Run full constraint test suite
- **Type**: auto
- **Description**: Build and run `cargo run --example test_ane_constraints`. This runs each test in a subprocess. Capture all stdout/stderr. Tests that fail will exit with non-zero codes — capture those error messages.
- **Verification**: All 30+ tests produce a result (OK or error message)
- **Done**: Complete results table with pass/fail for every test

### Task 2: Analyze results against Orion predictions
- **Type**: auto
- **Description**: Compare each test result against the expected outcome from the Orion paper. Flag surprises — tests that pass when Orion predicted failure (e.g., if `concat` works), or tests that fail unexpectedly. Identify which constraints are hard blockers vs. soft limits.
- **Verification**: Analysis document created with comparison table
- **Done**: Clear pass/fail/surprise categorization for all 20 Orion constraints

### Task 3: Update ane_constraints.md with empirical findings
- **Type**: auto
- **Description**: Update `docs/ane_constraints.md` to add "Empirical Result" column/section for each constraint. Mark which constraints are confirmed, refuted, or uncertain on this hardware. Add any new constraints discovered during testing.
- **Verification**: Document updated with actual test results
- **Done**: Every constraint has an empirical result, not just Orion's claim

### Task 4: Commit constraint test results
- **Type**: auto
- **Description**: Commit the test results documentation and any code fixes made during testing.
- **Verification**: Git commit created with clear message
- **Done**: Results committed

## Success Criteria

1. Every constraint test produces a clear PASS or FAIL result
2. Results documented with compile times, eval times, and error messages
3. `docs/ane_constraints.md` updated with empirical findings
4. Surprising results flagged for investigation
