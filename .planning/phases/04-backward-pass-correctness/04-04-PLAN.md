# Numerical gradient checking:
# For each weight w_i: dL/dw_i ≈ (L(w_i + eps) - L(w_i - eps)) / (2 * eps)
# Compare ANE analytical gradient against this numerical approximation.
# Use eps = 1e-3 for fp16 (larger than fp32's 1e-7 due to precision).
# Tolerance: ±5% relative error (fp16 has limited precision).
#
# Test approach:
# 1. Generate random weights and input
# 2. Run ANE backward MIL to get analytical gradients
# 3. For each weight, perturb by eps, run forward, compute numerical gradient
# 4. Compare analytical vs numerical: |analytical - numerical| / |numerical| < 0.05
#
# CPU forward pass needed: implement simple conv1x1 in pure Rust (matrix multiply)
# for the numerical gradient computation. This does NOT need ANE.
</context>

   - Create random input x, compute forward: h1=W1@x, h3=W3@x, silu=h1*sigmoid(h1), gate=silu*h3, y=W2@gate
   - Create d_out = ones (gradient of dummy loss w.r.t. output)
   - Pack input: [d_out, h1, h3] as fp16
   - Run ANE bwd_ffn_mil, get (dh1, dh3, dx) outputs
   - For dx: numerically verify by perturbing x, checking output change
   - Report max relative error and pass/fail per gradient
   - Print "OK numerical_ffn" if all within ±5%

3. Tests bwd_qkv_mil():

   - Create random dq, dk, dv and weights
   - Run ANE bwd_qkv_mil, get dx
   - Numerically verify by perturbing dq/dk/dv
   - Print "OK numerical_qkv" if within ±5%

4. Summary: print overall pass/fail count
