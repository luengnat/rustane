  <name>Task 1: Implement bwd_ffn_mil() in programs.rs</name>
  <files>src/mil/programs.rs</files>
  <action>
Port `gen_ffn_bwd()` from stories_mil.h (lines 111-146) to Rust as `bwd_ffn_mil()`.

Function signature: `pub fn bwd_ffn_mil(seq_len: usize, dim: usize, hidden_dim: usize) -> String`

Follow the reference code structure EXACTLY but with these substitutions:
1. `sub(x, y)` → `add(x, mul(y, const(-1.0)))` — sub is rejected on ANE
2. Output `concat(dx, dh1, dh3)` → THREE separate return outputs (multi-output) — concat is rejected
3. All conv ops use CONV_CONST pattern (pt, st, pd, dl, gr) from stories_mil.h

The MIL program must:
- Accept single fp16 input: `[1, DIM+2*HIDDEN, 1, SEQ]` (packed forward taps)
- Use `slice_by_size` to extract dffn (first DIM channels), h1 (next HIDDEN), h3 (next HIDDEN)
- Load transposed weight matrices W2t `[HIDDEN, DIM, 1, 1]`, W1t `[DIM, HIDDEN, 1, 1]`, W3t `[DIM, HIDDEN, 1, 1]` via BLOBFILE
- Compute dsilu = conv(W2t, dffn)
- Compute sig = sigmoid(h1)
- Compute oms = add(const(1.0), mul(sig, const(-1.0)))  [sub decomposition]
- Compute homs = mul(h1, oms)
- Compute brk = add(const(1.0), homs)
- Compute dsd = mul(sig, brk)
- Compute t1 = mul(dsilu, h3)
- Compute dh1 = mul(t1, dsd)
- Compute slh = mul(h1, sig)
- Compute dh3 = mul(dsilu, slh)
- Compute dx1 = conv(W1t, dh1)
- Compute dx3 = conv(W3t, dh3)
- Compute dx = add(dx1, dx3)
- Return three outputs: (dh1, dh3, dx) — alphabetical order for multi-output

Use the EXACT ANE MIL syntax from stories_mil.h: `name=string(...)`, `val=fp16(...)`, `const()[name=string(...), val=...]`, etc.

Also add `bwd_ffn_compile_request()` helper following the pattern of `conv1x1_compile_request()`.

Add the function to the public exports in `src/mil/mod.rs`.
